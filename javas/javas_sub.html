    Overview

    • 목표

    Problems
      & Solution

    • 1. position과 z-index 사용으로 인한 hover 이벤트 발생하지 않는 문제

      스크롤 시 상위 영역들에 의해 footer 영역이 가려지도록 시각적 효과를 나타내고 싶었으나 a 태그를 사용한 요소에 hover 이벤트가 적용되지 않는 문제점 발생

      일부 영역에 z-index 사용으로 인한 문제였으며 footer 영역 보다 a 태그에 z-index 값을 더 크게 줬더니 hover 이벤트 적용되어 해결함

    • 2. 이미지 박스 hover 시 텍스트와 이미지의 밝기가 함께 변화하는 문제

      메인 페이지에서 이미지 박스에 hover 시 백그라운드 이미지만 밝아지는 이벤트가 발생하는데 텍스트도 함께 바뀌는 문제가 발생함
      opacity 속성과 background: rgba();를 이용하였지만 지속적인 문제 발생

      배경을 혼합했을 때의 상태를 설정할 수 있는 background-blend-mode : multiply; 을 적용하여 문제를 해결함

    • 3. Animation 적용 가로로 흘러가는 텍스트가 넘치는 현상 발생

      넘치는 요소 제어를 위한 overflow: hidden을 사용해야 한다는 것을 알고 있었지만 값을 적용시켜도 요소들이 넘치는 문제가 해결되지 않음

      html 구조를 다시 살펴보니 계속해서 형제 요소에 적용했었고 부모 요소에 적용하니 올바르게 해결

    Review

    • 잘한 점

      퍼블리싱 가이드와 디자인 가이드를 제작하여 팀원들과의 협업이 원활하게 진행됨

      구현하기 어려워 보였던 스크립트들을 포기하지 않고 모두 구현함

      팀원들끼리 각자 맡은 부분이 아니더라도 어려운 부분이 있으면 서로 도와가며 협업함

    • 부족한 점

      서브 페이지 세 부분의 레이아웃 디자인이 같아서 더 다양한 디자인으로 했으면 좋았을 것 같다는 아쉬움이 남음

      예약 화면에서 예약 창이 잘 보이지 않아 좀 더 세심하게 신경 쓰지 못했다는 점

    • 프로젝트 완료 리뷰

      디자인부터 퍼블리싱까지 하는 팀 프로젝트는 처음이었으나 큰 문제 없이 잘 마무리되었다고 생각합니다.

      자바스크립트와 제이쿼리를 배우고 적용시켜야 했기 때문에 걱정이 앞섰지만 팀원들끼리 도와가며 모든 기능들을 구현할 수 있었습니다. 어려웠던 부분들도 있었지만 포기하지 않고 노력했기 때문에 나온 결과물이라고 생각합니다.

      또 개인적으로 제가 부족했던 부분을 객관적으로 파악할 수 있었고 스스로 공부할 수 있는 계기가 되었습니다.

      또한 깃허브의 협업 기능을 사용하면서 충돌 상황을 자주 접하게 되었고 팀원들과 소통을 하고 효율적으로 파일 정리를 어떻게 해야 하는지 생각하고 배우는 계기였습니다.
